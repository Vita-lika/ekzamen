# ekzamen

## 1 БИЛЕТ

 1)  1:Информатика — это комплексная, техническая наука, которая систематизирует приёмы создания, сохранения, воспроизведения, обработки и передачи данных средствами вычислительной техники, а также принципы функционирования этих средств и методы управления ими. 

Разделы информатики:
-Теоретическая информатика. Раздел информатики, который активно использует математический аппарат для описания различных информационных процессов. 
-Вычислительная техника. Раздел, в котором выполняется разработка общих принципов построения вычислительных систем. 
-Программирование. Раздел информатики, который занимается разработкой системного и прикладного программного обеспечения. 
-Информационные системы. Составная часть информатики, отвечающая за анализ потоков информации, их оптимизацию, структурирование, принципы хранения и поиска информации. 
-Искусственный интеллект. Раздел информатики, в котором решаются вопросы различных наук : моделирование рассуждений, генерация новых знаний, перевод с одного языка на другой с помощью программного обеспечения и др.. 

2:Информация — это осознанные сведения, различные знания, которые выражены в сигналах, известиях, уведомлениях, об окружающем мире, являющиеся объектом хранения, передачи и использования. 

3:Источник информации — это то, что несёт любую информацию человеку. Любой предмет, явление или живое существо может быть источником информации, если на него обратили внимание. 

 2)  Свойства информации: достоверность, объективность, полнота, актуальность, понятность, доступность, релевантность, эргономичность. 

 3)  Разница между данными и информацией: данные представляют собой информацию, находящуюся в формализованном виде и предназначенную для обработки техническими системами. Информация в отличие от данных — это не всё, что мы знаем о предмете, а только то, что нам интересно, что можно хранить, накапливать, применять, передавать и т. д.

## 2 БИЛЕТ 

 1) 1:Носитель информации — это материальный объект, на котором фиксируется информация. Например, бумага, жёсткие диски, оптические диски, флеш-карты. 

2: Канал связи — технические устройства или среда для передачи сигналов от источника информации к приёмнику. Например, электромагнитные, звуковые и световые волны. 

3:Сигнал — это изменение во времени некоторой физической величины, которое служит для отображения, регистрации и передачи сообщений. Сигналы могут быть аналоговыми (непрерывный поток информации) и дискретными (прерывистый). 

4:Сообщение — это информация, переданная по каналу связи. Например, Сергей написал сообщение Тане в социальной сети, в котором сообщил о времени начала трансляции на своём канале. 

5:Избыточность сообщения — это привычное проявление избыточности, например, извержение вулкана, речь политика, поворот руля, смена парадигмы, мысль или слово. 

 2) Этапы информационных процессов:
-Сбор информации. Поиск и отбор нужных данных из различных источников. 
-Представление информации. Преобразование информации в форму, которая удобна для её понимания и использования. 
-Обработка информации. Получение новых данных из тех, что уже имеются. Например, запись голоса на диктофон — звуковая информация преобразовалась в цифровую в виде кода. 
-Хранение информации. Фиксирование данных на материальном носителе. Например, записи в школьной тетради или публикация поста в социальной сети. 
-Передача информации. Перемещение данных от источника к приёмнику по каналу передачи. Например, Лена звонит маме по мобильному телефону, чтобы сообщить, что она получила сегодня пятёрку за контрольную работу по информатике.

## 3 БИЛЕТ

 1) 1:Бит в теории информации — количество информации, необходимое для различения двух равновероятных сообщений. В вычислительной технике битом называют наименьшую «порцию» памяти компьютера, необходимую для хранения одного из двух знаков «0» и «1», используемых для внутримашинного представления данных и команд. 

2:Формула Хартли показывает связь между количеством информации и количеством возможных вариантов. Количество информации I, содержащееся в выбранном сообщении, определяется как двоичный логарифм N, где N — количество символов в используемом алфавите (мощность алфавита), K — длина сообщения (количество символов в сообщении). 

3:Формула Шеннона учитывает возможную неодинаковую вероятность сообщений в наборе. Количество информации I определяется как сумма произведений вероятностей сообщений на логарифмы по основанию 2 этих вероятностей. 

4:Скорость передачи данных — это объём данных, передаваемых за единицу времени.  

2) Единицы информации и их соотношение:
-1 байт — 8 бит; 
-1 килобайт (Кб) — 1024 байта; 
-1 мегабайт (Мб) — 1024 Кб; 
-1 гигабайт (Гб) — 1024 Мб.

## 4 БИЛЕТ

1) 1:Линейные структуры данных — это упорядоченные структуры, в которых адрес элемента однозначно определяется его номером. Примеры: массивы, списки, стеки и очереди. 

2:Иерархические структуры данных — это структуры, в которых элементы более низкого уровня входят в элементы более высокого уровня. Пример: книги, разбитые на части, разделы, главы, параграфы. 

3:Ассоциативные структуры данных (карты) — это структуры, похожие на массивы, но вместо упорядоченных числовых индексов в них используются «ключи» — заданные пользователем числа или строки. Пример: телефонный справочник, где ключ — это имя, по которому человек ищет нужный номер, а значение — номер телефона. 

4:Специальные структуры данных — это сложные и специализированные устройства данных, которые обеспечивают эффективное хранение, поиск и манипулирование информацией в информатике и программировании. 

5:Табличные структуры данных — это структуры, в которых элементы данных определяются адресом ячейки, который состоит не из одного параметра, как в списках, а из нескольких. Пример: таблица умножения, где адрес ячейки определяется номерами строки и столбца. 

6:Последовательные структуры данных — это структуры, в которых элементы идут один за другим и доступны в определённой последовательности. Пример: слово, где данные — буквы. 

## 5 БИЛЕТ 

 1) 1:Компьютер — это универсальное электронное программно-управляемое устройство для создания, обработки, накопления и передачи информации. 

2:Программное обеспечение (софт) — это совокупность всех программ, необходимых для работы компьютера. 

3:«Железо» в программировании — это аппаратная часть компьютера, то есть все металлические или пластмассовые коробки, экраны, микросхемы и провода. 

4:Операционная система — комплекс программ, обеспечивающих согласованное функционирование всех устройств компьютера и предоставляющих пользователю доступ к ресурсам компьютера. 

5:Периферия в программировании — это совокупность внешних устройств, которые как-либо подключены к компьютеру. К ним относятся, например, устройства ввода и вывода информации (клавиатура, компьютерная мышь, принтер и т.д.).

 2) Основные компоненты компьютера: процессор, материнская плата, внутренняя (основная) память, внешняя память, устройства ввода, устройства вывода. 

 3) Разница между системным программным обеспечением и прикладным программным обеспечением в том, что системное программное обеспечение обеспечивает взаимодействие человека, всех устройств и программ компьютера, определяет на компьютере системную среду и правила работы в ней. Прикладное программное обеспечение — это комплекс готовых программ, предназначенных для выполнения определённых пользовательских задач и рассчитанные на непосредственное взаимодействие с пользователем.атное обеспечение — это материальная часть компьютера (его составные части и узлы).

## 6 БИЛЕТ 

 1) 1:Язык — это знаковая система для представления и передачи информации. 

2:Язык программирования — это формальный язык, к которому относятся системы математических, химических символов, нотная грамота, азбука Морзе и многие другие. 

3:Естественный язык — это язык, используемый для общения людей. Примеры естественных языков: русский, английский, китайский, французский и пр.. 

4:Формальный язык — это язык профессионального общения или определённой области знаний. Особенностью формальных языков является то, что все правила в них задаются в явной форме, что обеспечивает однозначность записи и восприятия сообщений на этих языках.

5:Слово — это последовательность символов алфавита, кодирующая информацию. 

6:Алфавит — это упорядоченный набор символов или сигналов, который составляет основу языка. 

 2) Сопоставление естественных и формальных языков:
Естественные языки развивались веками и служили для общения людей между собой. Формальные языки разрабатываются для специальных применений. 
Естественные языки характеризуются широкой сферой применения, наличием большого количества правил, гибкостью и динамичностью. Формальные языки строятся в соответствии с чёткими правилами, обеспечивая непротиворечивое, точное и компактное отображение свойств и отношений изучаемой предметной области.

## 7 БИЛЕТ

 1) 1:Кодирование — это процесс перевода информации из формы, понятной человеку (текст, изображения, видео и т. д.), в некоторый код. 

2:Код — это условные знаки для представления информации. 

3:Двоичное кодирование — метод хранения и отображения символов с применением двух знаков: нулей и единиц. Это универсальный способ отображения любых информационных сведений (текстовых сообщений, картинок, звуковых и видеоматериалов). 

4:Декодирование — это процесс перевода информации из кода в форму, понятную человеку. 

5:Кодирование символов — присвоение любому символу текста кода из кодировочной системы. 

6:Unicode — символьная таблица, которая используется в современных компьютерах. Это официальный стандарт, который включает в себя практически все знаки существующих на письме языков. 

7:ASCII (American standard code for information interchange) — это кодовая таблица, которая была создана в 1963 году в США и стала первым мировым стандартом. В ней находились коды для 128 символов, соответственно, один символ весил 7 бит. 

 2) Стандарт ASCII включает следующие виды кодировок:
-ASCII-8. Расширяет набор символов, добавляя дополнительные символы. 
-ASCII-16. В этой кодировке добавлены дополнительные символы, которые расширяют возможности кодирования. 
-ASCII-32. Предоставляет ещё больше символов для различных приложений. 
-ASCII-64. Набор символов увеличивается, поддерживая массив символов и международные символы. 
-ASCII-128. Расширенный набор ASCII-128 дополняет спектр из 256 символов, включая широкий спектр символов. 

Особенности кодировки ASCII:
Структура таблицы. Первые 32 кода зарезервированы под управляющие символы. Они не отображаются на экране, но играют важную роль в управлении потоками данных и форматированием текста. 
Ограниченный набор символов. ASCII может представлять только 128 символов (7-битная кодировка) или 256 символов (8-битная кодировка). 
Отсутствие поддержки нелатинских символов. ASCII не обеспечивает поддержку символов за пределами английского алфавита. 

## 8 БИЛЕТ

 1) 1:Система счисления — это совокупность правил записи чисел посредством конечного набора символов (цифр). 

2:Унарная система счисления — простейшая и самая древняя система, в которой для записи любых чисел используется всего один символ — палочка, узелок, зарубка, камушек. 

3:Непозиционная система счисления — система, в которой значение цифры не зависит от её позиции в числе. Пример — римская система записи чисел. 

4:Позиционная система счисления — система, в которой значение цифры меняется в зависимости от того, в каком месте она находится. Примеры позиционных систем счисления: десятичная, двоичная, восьмеричная, шестнадцатеричная. 

5:Основание системы счисления — количество различных цифр, используемых в этой системе. 

 2) Двоичная и десятичная системы счисления имеют разные основания. В десятичной системе основание 10 — в ней всего 10 цифр (0, 1, 2, ..., 9). В двоичной системе счисления основание 2, потому что цифр для записи чисел всего две: 0 и 1. 

Восьмеричная система счисления имеет основание 8. Это значит, что от 0 до 7 цифры идут как обычно, а когда число доходит до 8, начинается другой разряд и число записывается как 10. Соответственно, число 9 будет записываться как 11, а число 10 — как 12. 

Шестнадцатеричная система счисления имеет основание 16. Оно больше десяти, поэтому в алфавите появляются дополнительные цифры, которые обозначают буквами. От 0 до 9 цифры идут как обычно. Но на десяти разряд ещё не меняется, поэтому для обозначения десятки нужна новая цифра. В качестве этой «цифры» используют латинскую букву A. 

Принцип перевода числа из одной системы счисления в другую заключается в следующем:
-Разделить число на основание системы, в которую переводится число. 
-Выписать остаток от деления. 
-Если полученное частное больше или равно основанию системы, в которую переводится число, перейти к шагу 1. 
-После получения частного меньше основания системы, выписать частное. 
-Записать по порядку все выписанные числа от последнего к первому. 

## 9 БИЛЕТ

 1) 1:Растр — это графическая информация, формируемая из определённого количества строк, содержащих определённое количество точек. 

2:Векторный рисунок — это изображения, состоящие из множества отдельных, масштабируемых объектов (линий и кривых), которые определены с помощью математических уравнений. 

3:Пиксель — минимальный участок изображения, для которого независимым образом можно задать цвет. 

4:Дискретизация — процесс разбиения на отдельные элементы (точки или пиксели), причём каждый элемент имеет свой цвет (красный, синий, зелёный и т.д.). 

5:Разрешение — это число пикселей в изображении на единицу длины. Оно является мерой чёткости деталей растрового изображения и обычно обозначается как dpi (точек на дюйм) или ppi (пикселей на дюйм). 

6:RGB — формат цвета, который выделяет для каждого цветного квадрата 3 байта: один для красных оттенков, второй для зелёных оттенков, третий для синих оттенков. 

7:CMY — система цветопередачи, в которой каждый элемент имеет свой цвет (красный, синий, зелёный и т.д.)

 2) Глубина цвета — это количество бит, отводимых на кодирование одного пиксела. Чем больше цветовая глубина, тем больше цветов воспроизводится устройством (монитором, сканером, принтером) и тем качественнее смотрится изображение. 

Перевод из RGB-модели в CMY осуществляется следующим образом: алгоритм перевода прост, при этом предполагается, что цвета RGB находятся в интервале [0;1]. Для получения чёрного цвета в модели CMY необходимо смешать голубой, пурпурный и жёлтый в равных пропорциях. 

-Принцип формирования цветов в RGB-модели основан на том, что все цвета получаются путём смешения трёх базовых (красного, зелёного и синего) цветов в различных пропорциях. 

-Принцип формирования цветов в CMY-модели заключается в том, что цвета создаются путём наложения голубого, пурпурного и жёлтого цветов на белый лист бумаги, а также чёрного цвета для улучшения контраста и глубины цвета. Каждый цвет создаётся путём комбинации этих четырёх основных цветов в различных пропорциях. 

## 10 БИЛЕТ 

 1) 1:Логика — наука, изучающая законы и формы мышления. 

2:Формальная логика — наука о законах и операциях правильного мышления. Согласно её основному принципу, правильность рассуждения определяется только его логической формой и не зависит от конкретного содержания входящих в него утверждений. 

3:Высказывание — повествовательное предложение, о котором можно сказать, истинно оно или ложно. Высказывание может принимать только одно из двух логических значений — истинно (1) или ложь (0). 

4:Алгебра логики — это математический аппарат, с помощью которого записывают (кодируют), упрощают, вычисляют и преобразовывают логические высказывания. 

 2) Логические операции удобно производить с помощью двоичной системы счисления, потому что помимо выполнения математических действий с её помощью можно вычислять истинность и ложность высказываний, обозначая истинные как 1, а ложные как 0.
 
 3) Основные логические операторы:
-AND (И). Обозначается знаком & и умножает биты друг на друга. Значение истинно, когда оба бита равны единице. 
-OR (ИЛИ). Обозначается знаком | и складывает два бита, не превышая разряда числа. Значение истинно, когда один или два бита равны единице. 
-XOR (исключающее ИЛИ). Обозначается как |. Истина, если только один бит равен единице. 
-NOT (НЕ). Инвертирует бит: 0 в 1 и наоборот.

## 11 БИЛЕТ

 1) 1:Таблицы истинности — это математические таблицы, используемые для определения функционирования логических операций в булевой алгебре. Каждая таблица истинности представляет все возможные входные комбинации для данной логической операции и соответствующие им результаты. 

2:Логическая функция — это функция, определённая на множестве истинностных значений (И, Л) и принимающая значения в этом множестве. 

3:Конъюнкция (логическое умножение, логическое «И») — это сложное логическое выражение, которое считается истинным в том и только том случае, когда оба простых выражения являются истинными, во всех остальных случаях данное сложное выражение ложно. Пример: высказывание «В комнате включён свет» (истина) и «На улице идёт снег» (ложь). Тогда конъюнкция этих утверждений будет ложной: «В комнате включён свет или на улице идёт снег». 

4:Дизъюнкция (логическое сложение, логическое «или») — это сложное логическое выражение, которое истинно, если хотя бы одно из простых логических выражений истинно и ложно тогда и только тогда, когда оба простых логических выражения ложны. Пример: «В комнате включён свет или на улице идёт снег». Поскольку первое утверждение истинно (в комнате действительно включён свет), даже несмотря на то, что второе ложно (на улице не идёт снег), вся дизъюнкция будет истинной. 

5:Отрицание (инверсия, логическое «НЕ») — это сложное логическое выражение, если исходное логическое выражение истинно, то результат отрицания будет ложным, и наоборот, если исходное логическое выражение ложно, то результат отрицания будет истинным. Пример: высказывание «Сегодня идёт дождь». Тогда инверсия этого утверждения будет истинной: «Сегодня не идёт дождь». 

6:Импликация (логическое следование) — это сложное логическое выражение, которое истинно во всех случаях, кроме как из истины следует ложь. То есть данная логическая операция связывает два простых логических выражения, из которых первое является условием (А), а второе (В) — следствием. Пример: «Если я выпил чашку кофе утром, то я чувствую себя бодрым». В данном случае первое утверждение истинно (вы действительно выпили чашку кофе утром), а второе ложно (вы не чувствуете себя бодрым, несмотря на выпитый кофе). 

7:Эквивалентность (логическая равнозначность) — это сложное логическое выражение, которое является истинным тогда и только тогда, когда оба простых логических выражения имеют одинаковую истинность. Пример: «В комнате включён свет или на улице идёт снег». В данном случае, поскольку одно из утверждений истинно, вся эквиваленция также становится истинной.

 2) Разница между системными и прикладными программами заключается в их назначении и роли в компьютерной системе.
-Системные программы расширяют возможности базового программного обеспечения и организуют более удобную среду работы пользователя. Они выполняют различные вспомогательные функции, например, создание копий, выдачу справочной информации, проверку работоспособности устройств ПК. К системным программам относятся драйверы, оболочки, утилиты, упаковщики, антивирусные программы, программы для диагностики ПК, управления локальной сетью и другие. 

-Прикладные программы решают задачи конечного пользователя. Они запускаются по запросу пользователя и работают на платформе, предоставляемой системным программным обеспечением. Для написания прикладных программ используются языки высокого уровня. К ним относятся текстовые редакторы, табличные процессоры, системы динамических презентаций, управления базами данных, графические редакторы и другие. 

Необходимость классифицировать программы на пользовательские и системные связана с тем, что у каждого из этих видов программ своя уникальная роль: системное программное обеспечение гарантирует бесперебойное управление оборудованием, а прикладное — что пользователь может выполнять любую конкретную задачу плавно и непосредственно, без каких-либо помех.

 3) Windows, Linux и macOS — три семейства операционных систем для ПК. У каждого из них есть свои особенности и преимущества. 

Сравнительный анализ можно провести по следующим параметрам:
-Интерфейс и дизайн. Windows: удобный интерфейс со знакомой раскладкой, панелью задач, меню «Пуск» и оконными приложениями. Linux: дистрибутивы предлагают множество интерфейсов пользователя (среды рабочего стола) — GNOME, KDE и другие. macOS: элегантный и привлекательный интерфейс, известный своим дизайном и интуитивностью навигации. 

-Совместимость с программным обеспечением. Windows: поддерживает большое количество коммерческого и проприетарного программного обеспечения. Linux: имеет надёжную экосистему программного обеспечения с открытым исходным кодом, предлагая широкий спектр бесплатных и открытых приложений. macOS: поддерживает значительное количество популярных приложений, в частности разработанных Apple и стороннего софта, оптимизированного для систем Mac. 

-Настраиваемость и гибкость. Windows: возможности настройки относительно ограничены по сравнению с macOS и Linux, но есть множество сторонних инструментов для настройки. Linux: славится гибкостью и потенциалом настройки. Пользователи могут изменить почти каждый аспект операционной системы, включая среду рабочего стола, внешний вид и поведение. macOS: пользователи могут персонализировать рабочие столы, настроить параметры, выбрать различные темы и визуальные модификации. Однако возможности настройки macOS обычно более ограничены, чем у Linux. 

-Безопасность и стабильность. Windows: за годы развития сделала значительные шаги в повышении безопасности, но остаётся основной целью для вредоносных программ и вирусов из-за своей популярности. Linux: часто считается более безопасным, чем Windows и macOS, благодаря надёжной системе разрешений, частым обновлениям безопасности и открытому исходному коду. macOS: имеет репутацию сильной безопасности из-за жёсткого контроля Apple над аппаратной и программной экосистемой.

## 15 БИЛЕТ 

1) 1:Топология сети — это способ описания конфигурации сети, схема расположения и соединения сетевых устройств.
   
2:Общая шина — это топология, в которой компьютеры (рабочие станции) подключены к одному кабелю с помощью специальных разъёмов. Чтобы сигнал не отражался от концов кабеля, их закрывают заглушками (терминаторами). Так существует всего одна линия связи, компьютеры передают данные по очереди. 

3: Терминатор — это начало, конец или прерывание процесса обработки данных или выполнения программы.

4:Звезда — топология, в которой все компьютеры сети присоединены к центральному узлу (обычно коммутатору), через который идёт весь обмен данными. Коммутатор передаёт принятый пакет только адресату, а не всем компьютерам в сети. 

5:Сетевой коммутатор (свитч) — это устройство, которое соединяет несколько компьютеров в одну локальную сеть и позволяет им обмениваться данными.  

6:Дерево — это нелинейная иерархическая структура данных, которая представляет собой совокупность элементов и отношений, образующих структуру в виде набора связанных узлов 

7:Кольцо — это топология, в которой каждый компьютер соединён с двумя соседними: от одного он только получает информацию, а другому только передаёт. Таким образом, пакеты движутся по кольцу в одном направлении.

8:WAP (Wireless Application Protocol) — это протокол беспроводной передачи данных. 

 2) Разница между коммутатором и концентратором заключается в способе передачи данных. 
-Концентратор (также известный как хаб) рассылает трафик от центральной машины (ПК, сервера) сразу всем компьютерам. Конечный ПК или сервер самостоятельно определяет, является ли он адресатом или нет. Если устройство не выступает таковым, то пакет попросту уничтожается.

Коммутатор знает, какие устройства соединены с его портами, и направляет пакеты только на необходимые порты. Это даёт возможность одновременно работать с несколькими портами, расширяя тем самым полосу пропускания.

 4)Провайдер — это телекоммуникационная компания, предоставляющая услуги доступа к сети Интернет и иные связанные с Интернетом услуги.

## 17 БИЛЕТ 

 1) 1:TCP/IP (Transmission Control Protocol/Internet Protocol) — это основной набор протоколов, используемых в интернете и локальных сетях. Обеспечивает надёжную передачу данных и маршрутизацию пакетов между устройствами. TCP отвечает за установление соединения и контроль ошибок, а IP — за адресацию и маршрутизацию пакетов. 

2:IP-адрес — это уникальный номер устройства, подключённого к интернету или локальной сети, чаще всего компьютера, сервера или мобильного гаджета.

3:IPv4 — это четвёртая версия интернет-протокола IP, созданная в 1981 году. Является основным протоколом коммуникации в Интернете. Использует 32-битное адресное пространство, общее количество IP в Сети — около 4,3 миллиарда. 

IPv6 — шестая версия интернет-протокола IP, созданная в декабре 1995 года. Протокол может предоставить практически бесконечное количество адресов для устройств и решает проблему нехватки IP-адресов. В отличие от IPv4, типичный адрес IPv6 состоит из 128 бит.

4:HTTP/HTTPS (HyperText Transfer Protocol и HTTPS (HTTP Secure)) — это протоколы, используемые для передачи данных в веб-приложениях. HTTPS обеспечивает безопасное соединение с использованием шифрования. HTTP используется для передачи гипертекстовых документов, таких как веб-страницы, а HTTPS добавляет уровень безопасности, защищая данные от перехвата и подмены. 

5:FTP (File Transfer Protocol) — это протокол для передачи файлов между устройствами в сети. Он позволяет загружать и скачивать файлы с удалённых серверов. FTP поддерживает аутентификацию пользователей и управление доступом к файлам, что делает его удобным инструментом для обмена большими объёмами данных. 

6,7:SMTP/IMAP/POP3 (Simple Mail Transfer Protocol, IMAP (Internet Message Access Protocol) и POP3 (Post Office Protocol)) — это протоколы, используемые для передачи электронной почты. SMTP отвечает за отправку писем, а IMAP и POP3 — за получение. IMAP позволяет хранить письма на сервере и синхронизировать их между устройствами, тогда как POP3 загружает письма на локальное устройство и удаляет их с сервера. 

8:DNS (Domain Name System) — это протокол, который преобразует доменные имена в IP-адреса. Он позволяет пользователям использовать удобные для чтения имена вместо числовых IP-адресов. DNS играет ключевую роль в работе интернета, обеспечивая быстрый и надёжный доступ к веб-ресурсам и другим сетевым сервисам. 

9:SSL (Secure Sockets Layer) и TLS (Transport Level Security) — криптографические протоколы, обеспечивающие защищённую передачу данных в компьютерной сети. 

10:DHCP (Dynamic Host Configuration Protocol) — это протокол, который автоматически назначает IP-адреса устройствам в сети. Он упрощает управление сетевыми настройками, позволяя администраторам централизованно управлять адресным пространством и минимизировать вероятность конфликтов IP-адресов.

 2) Уровни протоколов в модели OSI и примеры для каждого из них:

-Физический уровень. На этом уровне протоколы переводят информацию в сигнал и обратно в биты, отвечают за её передачу по кабелю или в эфир. Примеры протоколов этого уровня — Bluetooth, Wi-Fi или GSM, по которому реализована мобильная связь. 

-Канальный уровень. Уровень описывает взаимодействие между устройствами по физическому каналу — например, как принимать и интерпретировать сигналы. Протоколы второго уровня получают на вход биты и упаковывают их в кадры — фрагменты данных, а ещё проверяют информацию на ошибки. Примеры протоколов на этом уровне — Ethernet, который отвечает за передачу данных по витой паре, или FDDI — он работает с оптоволоконным каналом. 

-Сетевой уровень. Третий уровень отвечает за путь, по которому будут передаваться данные. Протоколы определяют оптимальный маршрут для передачи данных, соотносят между собой физические и логические адреса устройств. Известнейший пример протокола сетевого уровня — IP, благодаря которому фактически появился интернет. Устройства сетевого уровня — это маршрутизаторы, роутеры. 

-Транспортный уровень. Четвёртый уровень задаёт «правила дорожного движения» при передаче данных. Протоколы транспортного уровня формируют очередь, объединяют короткие пакеты данных в один большой или разделяют слишком большие пакеты на несколько маленьких. Ещё протоколы отвечают за надёжность передачи сообщений, чтобы данные не «потерялись» и не повредились по пути. На этом уровне работают протоколы TCP, UDP и другие — они тоже используются в интернет-сети. 

-Сеансовый уровень. Отвечает за сеанс связи между разными программами — в рамках сеанса два абонента на разных устройствах обмениваются информацией. На сеансовом уровне находятся протоколы, которые отвечают за начало и конец сеанса, его поддержание, когда обе программы «молчат», и за синхронизацию — чтобы сообщения от обеих сторон передавались вовремя. Примеры протоколов этого уровня — NetBIOS (Network Basic Input/Output System), который обеспечивает сеансовые услуги для приложений в локальных сетях, и RPC (Remote Procedure Call), позволяющий программам вызывать функции на удалённых узлах. 

-Представительный уровень. Отвечает за преобразование данных между форматом, используемым приложением, и форматом, используемым сетью. Он обеспечивает кодирование, шифрование и сжатие данных. Примеры протоколов этого уровня — SSL/TLS (Secure Sockets Layer/Transport Layer Security), который обеспечивает шифрование данных для безопасной передачи, и MIME (Multipurpose Internet Mail Extensions), определяющий формат данных для передачи мультимедийных сообщений. 

-Прикладной уровень. Он отвечает за то, как с сетью взаимодействуют пользовательские приложения: браузер, клиент электронной почты и так далее. Протоколы прикладного уровня регулируют доступ приложений к сети, помогают передавать служебные данные программ и пересылать сообщения об ошибках. Примеры протоколов этого уровня — HTTP (Hypertext Transfer Protocol) для браузеров и SMTP (Simple Mail Transfer Protocol) для электронной почты.

 3) IP-адреса — это уникальные числовые адреса компьютеров в сети. Они представляют собой строку из четырёх чисел, разделённых точками. В этих числах зашифрована информация о местоположении сервера с файлами сайта. 

Доменные имена — это уникальная последовательность символов (букв, цифр, точек и иногда дефисов), благодаря которой пользователи могут находить веб-ресурсы в глобальной сети. Каждый символьный адрес неповторим и может быть «привязан» только к одному сайту.

 1) 1:Сортировка пузырьком — один из простейших методов сортировки. Заключается в постепенном смещении элементов с большим значением в конец массива. Элементы последовательно сравниваются попарно, и если порядок в паре нарушен — меняются местами. 
Пример реализации: процедура BubbleSort(a: array of integer). 

2:Сортировка «методом камня» — это алгоритм, при котором самый «тяжёлый» элемент опускается в конец массива. 
Примерный код на Python для реализации такой сортировки: 

li = [5, 2, 7, 4, 0, 9, 8, 6]
n = 1
while n < len(li):
    for i in range(len(li)-n):
        if li[i] > li[i+1]:
            li[i], li[i+1] = li[i+1], li[i]
    n += 1
print(li)

3:Сортировка выбором — это гибрид между пузырьковой и сортировкой вставками. Алгоритм ищет наименьший элемент в текущем списке и производит обмен его значения со значением первой неотсортированной позиции. То же самое происходит со вторым элементом с наименьшим значением. Цикл повторяется до тех пор, пока все элементы не займут нужную последовательность. 

4:Быстрая сортировка считается одним из самых быстрых алгоритмов сортировки. Работает по принципу «разделяй и властвуй». Алгоритм состоит из трёх шагов: выбор опорного элемента из массива, перераспределение элементов в массиве таким образом, что элементы меньше опорного помещаются перед ним, а больше или равные — после, рекурсивное применение первых двух шагов к двум подмассивам слева и справа от опорного элемента. 

5:Сортировка слиянием следует принципу «разделяй и властвуй», согласно которому массив данных разделяется на равные части, которые сортируются по-отдельности. После они сливаются, в результате получается отсортированный массив. 

6:Сортировка Шелла — алгоритм сортировки, являющийся усовершенствованным вариантом сортировки вставками

7:Поразрядная сортировка — алгоритм сортировки, который выполняется за линейное время. Исходно предназначен для сортировки целых чисел, записанных цифрами. Но так как в памяти компьютеров любая информация записывается целыми числами, алгоритм пригоден для сортировки любых объектов, запись которых можно поделить на «разряды», содержащие сравнимые значения, например, строки, и вообще любые данные, представленные в виде набора байтов. 

8:Сортировка подсчётом (англ. counting sort) — алгоритм, который используется, когда элементы могут принадлежать только какому-то небольшому множеству. 

9:Пирамидальная сортировка выстраивает данные в виде двоичного дерева (двоичной кучи). Наибольший/наименьший элемент (в зависимости от типа) располагается в корне дерева. Он меняется местами с последним элементом кучи и помещается в конец массива. Размер кучи уменьшается на 1, после чего она перестраивается. Цикл повторяется, пока размер кучи больше 1.

 2) Работа с файлами в языке программирования реализуется следующим образом: перед работой с информацией в файле его нужно открыть. После этого можно выполнять операции чтения из файла и записи в файл. По окончании работы файл нужно закрыть. 

Примеры работы с разными типами файлов:
Для типизированных файлов. Процедура считывания read считывает один или несколько компонентов файла и присваивает значение некоторой переменной. Формат процедуры: Read(f,v1,v2,…,vn), где f — файловая переменная, v1,v2,…vn — имена переменных, которым будут присвоены данные из файла. Запись в файл осуществляется процедурой write. Формат процедуры: Write(f,v1,v2,…vn), где f — файловая переменная, v1,v2,…vn — имена переменных, значения которых записывается в файл. 

Для текстовых файлов. Текстовый файл представляет собой последовательность строк разной длины, состоящих из символов. К элементам текстового файла возможен только последовательный доступ, начиная с первого. Формат процедуры считывания из текстового файла: Read(f,v1,v2,…vn), где f — файловая переменная, v1,v2,…vn — имена переменных, которым будут присвоены данные из файла. Формат процедуры записи в текстовый файл: Write(f,v1,v2,…vn), где f — файловая переменная, v1,v2,…vn — имена переменных, значения которых записывается в файл. 

Для нетипизированных файлов. Для них при объявлении не определяется тип компонентов. Это позволяет получать доступ к файлам с любой структурой. При открытии файла (процедурами reset или rewrite) необходимо только указать размер элементов (записей), которыми будет проводиться обмен с файлом. При этом файл трактуется как последовательность этих записей (то есть элементов произвольного типа).
